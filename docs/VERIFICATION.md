# Verification

この文書は、修正後に確認する内容をまとめたチェックリストです。

## Baseline Verification

すべてのコード変更後に最低限確認する項目:

- Godot headless 起動でエラーが出ない。
- `res://scenes/title.tscn` がメインシーンとして維持されている。
- Autoload の `GameConfig` と `SoundManager` が有効。
- 起動時にゲームハブが表示される。
- Shengji / Tractor のゲーム開始ができる。
- Godot が参照するアセットパスが壊れていない。

推奨コマンド:

```sh
/Applications/Godot.app/Contents/MacOS/Godot --headless --path . --quit
```

## Shengji Smoke Test

Shengji 関連を触った場合:

- `scenes/shengji/main.tscn` を起動できる。
- 配札が開始される。
- 人間プレイヤーの手札が表示される。
- カードをクリックして選択・解除できる。
- 入札UIが表示される場面で操作できる。
- 底札フェーズで8枚選択が要求される。
- プレイフェーズで不正な出し方が拒否される。
- トリック終了時に得点と勝者メッセージが更新される。
- ラウンド終了またはゲーム終了UIが壊れていない。

## UI Verification

UIを触った場合:

- 1280x720 相当で文字やボタンが重ならない。
- 1600x900 相当で余白が不自然に広がりすぎない。
- 日本語、英語、中国語で主要ラベルがはみ出さない。
- カード選択時の持ち上がり、hover、z-index が不自然にならない。
- ボタンの有効/無効状態がフェーズと一致している。

## Localization Verification

`GameConfig.TEXT` を触った場合:

- 追加したキーが `en`、`ja`、`zh` のすべてに存在する。
- 既存キーの意味を変えていない。
- `%s`、`%d` などのフォーマット指定子が3言語で一致している。
- 長い翻訳でもUIが破綻しない。

## Asset Verification

カード、音声、動画などを触った場合:

- 共通カード画像は `assets/common/cards/` にある。
- `Card.get_card_name()` が生成するファイル名と画像名が一致する。
- `card_back.png` が存在する。
- 音声ファイルがなくても `SoundManager` のフォールバック音でエラーにならない。
- Godot import ファイルはエディタまたはGodot実行で生成されたものを使う。

## Rule Verification

Shengji ルールを触った場合:

- シングル、ペア、トラクター、投げ出しの判定を確認する。
- 切り札スート、レベルカード、ジョーカーの強さ順を確認する。
- リードに対して同スートを持っている場合、正しくフォロー強制される。
- 同スートがない場合、任意カードを出せる。
- 複数デッキの同一カードペアが正しくペア扱いされる。

## Risk Notes

現在のプロジェクトでは、UIとゲーム進行の多くがコードで動的生成されています。そのため、単純な構文チェックだけでは不十分です。UI、カードノードの親子関係、シグナル接続、フェーズ遷移は実行確認が必要です。
